News

03/28/2011 - News and updates can now be found on our slick Wordpress-driven blog.

03/24/2011 - These days it's hardly worth looking at a website unless it has slick rounded corners everywhere. They just scream Web 2.0, don't they? Well now we have them! Rejoice! I also hope to roll out a "Save Generated Dublin Core as File" feature this week. Everything works, I just want to test it a bit more first.

02/26/2011 - If you're the security conscious type, I've added the ability to use this site over an encrypted connection. Just go to https://www.dublincoregenerator.com (note the https://) and you'll be able to generate metadata with the added bonus of 256-bit encrypted privacy.

02/21/2011 - I've added what will hopefully be the first of a few minor validation features to the Advanced Generator. For the Coverage element, it will now restrict you to only the valid encoding schemes for Coverage/Spatial/Temporal depending on what you select. I'm hoping to add some additional features along these lines (e.g.: if you pick DCMI Type, checking to make sure you actually enter a valid DCMI Type). I've made a handful of other minor updates here and there to the text, style (mainly to get it looking decent in older versions of Internet Explorer), and formatting.

02/12/2011 - Huzzah! There are now separate Simple and Advanced Generators! The Simple Generator is just the basic 15 Dublin Core elements without any of the qualifiers or encoding schemes. The Advanced Generator is the generator that the site has had up until now with all of the valid qualifiers and encoding schemes included. I'll be curious to see which gets more use.

02/10/2011 - I've created a Twitter account for news and updates specific to this website. The username is dublincoregen. If you're curious about the excrutiating minutae of my own life, feel free to still follow me personally at nicksteffel.

02/09/2011 - I added a quick little thing that's been on my list for awhile today: a favicon! It's nothing fancy, but hopefully it'll make the site a little easier to spot in bookmark lists or tabbed browsers.

02/06/2011 - Just as an update, I have a version of the non-qualified Dublin Core generator working. It will be another week or two until it makes its way online since I'd like to redesign the site itself a little bit to better accomodate it.

01/25/2011 - I made a few minor changes to clean up the site's code earlier today. I also did something I probably should have done a long time ago: created a block of Dublin Core code for this site...using my site! It was uniquely satisfying.

01/24/2011 - I had not noticed until today that my update from 12/24 had actually broken the tool for Internet Explorer users -- copies of elements were no longer appearing in the generated code. I have revised the backend code that handles this and it should now work across all browsers. Sorry about that!

01/23/2011 - Cheers to Makx Dekkers, who just so happens to be the Managing Director and CEO of the Dublin Core Metadata Initative, for pointing out a few issues with how I was generating the HTML and XHTML code. Elements that refer to a URL should now generate as a <link> tag instead of a <meta> tag. Qualified DC elements will now be named DCTERMS.<insert name> instead of DC.<element name>.<qualified element refinement>. Similarly, vocabulary encoding schemes will now appear as DCTERMS.<scheme>. I also made a few minor corrections that I noticed while I was tinkering around.

01/16/2011 - Cheers to visitor David Bigwood for emailing me about the tool's HTML capabilities. He helped point out that the DC-HTML guidelines actually provide two sets of sub-guidelines: one for plain HTML and one for XHTML. I have updated the tool to generate either HTML or XHTML and also added some additional output options similar to those available for generating XML. I would be extremely curious to hear whether people are still actively using DC-HTML often since, admittedly, I added it mostly as an afterthought and have not planned to do a lot of developing for it. Drop me a line and let me know.

01/10/2011 - Cheers to visitor Samuel Spencer for pointing out that the closing container tag was incorrectly generating as <\metadata>. This is now fixed.

01/08/2011 - The next improvement in the works is a way to toggle the generator between the standard 15 Dublin Core elements and the much larger set of Qualified Dublin Core elements. The current generator will become the Qualified Dublin Core generator and I'll be creating a second much simpler one with just the core 15 elements and no refinements and/or encoding schemes. I'm hoping it won't take very long to do since everything's (obviously) in place, I just need to perform a little surgery to whittle it down while making sure everything still works.

01/01/2011 - Happy New Year! I have finished adding the additional qualified elements! I also added a "license" refinement for the "rights" element which I had overlooked. Additionally, I added the Medical Subject Headings and National Library of Medicine Classification as valid encoding schemes for "subject" along with links in the left-hand resources box. I've also snuck in a few minor updates to the page layout over the past week.

12/24/2010 - Merry Christmas! I've made a backend update I've been wanting to do ever since this tool went live. Previously the tool would only recognize the first ten copies of a given element. Now it should be able to recognize as many as you can throw at it (well, there is a JavaScript memory limit at around 4.2 billion...so try to keep it under that). I also fixed a bug where additional copies of the "Description" element didn't have the nice big input box for entering information.

12/11/2010 - Added a link to Creative Commons to the resources section. Also started laying the groundwork for adding the handful of special qualified Dublin Core elements (Provenance, AccrualPolicy, etc.) which I will hopefully have in working order within a week or so.

12/7/2010 - Made several minor changes to how the site will appear for people using Internet Explorer. I had tried to use the CSSHover workaround to give IE users some cool effects on the form fields, but it was creating some odd issues with using the drop-down menus. I also made a CSS tweak for IE users so the checkboxes don't look ugly.

12/4/2010 - The site goes live! Thanks to 6sync for the awesome reasonably priced hosting.
